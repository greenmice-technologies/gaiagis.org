MCP は、ツールがエージェントに機能を公開する方法を標準化するという強力なアイデアです。運用環境では、優勝チームは MCP サーバーを **他の特権付き統合**と同様に扱います。

### 私たちが使用する設計ルール

- **デフォルトでの**最小特権**: 範囲が狭く、明示的な認証情報、可能な限り有効期間の短いトークン。
- **監査可能なアクション**: 「成功/失敗」だけでなく、意図、入力、結果をログに記録します。
- **バージョン付きコントラクト**: スキーマと機能の変更は明示的です。サイレント破損は受け入れられません。
- **障害の分離**: ツールの停止によって自動化面全体がダウンしてはなりません。

### 私たちが測定するもの

レイテンシーの分布、ツールごとのエラー バジェット、重要なワークフローの回帰スイート (特にエージェントが複数の通話を連鎖できる場合)。

エンタープライズ環境向けの MCP を評価している場合は、デモではなく、脅威のモデリングと運用上の所有権から始めてください。
